FAQ

Why host a static site?

Because your website are going to run with much lower latency and much higher uptime. In most of the cases we don't need a dinamyc website, for that exists tools like Hugo, Jekyll, wintersmith, Sculpin, etc.

Why a service for host static sites?

All that tool cited before are awesome, but there is place for solutions of the problem in a high level, open sourced and using Golang. As a project for a hackaton, we thought that it could be greate experience for the team.

How hugoku works?

Hugoku it's an automated tool for creating and publishing static sites with the Hugo technology, and storaged in github as a repository.

In case of future, what will be the not hackathon features?

  • Public subdomain for static websites
  • Custom domain configuration
  • Response in http2 protocol
  • Automated tool for publish static websites in the diferents cloud providers like Amazon AWS, Google App Engine, Digital Ocean, etc
